This section covers the technical architecture and implementation details of the Vector frontend.

## Tech stack

| Layer | Technology |
|---|---|
| **Framework** | React 19 with TypeScript |
| **Routing** | React Router 7 |
| **UI Components** | shadcn/ui (Radix + Tailwind CSS 4) |
| **Data Fetching** | React Query (TanStack Query) |
| **HTTP Client** | Axios (with JWT interceptor) |
| **Data Tables** | TanStack React Table |
| **Rich Text Editor** | TipTap (with tiptap-markdown for serialization) |
| **Drag & Drop** | @dnd-kit |
| **Toasts** | Sonner |
| **Icons** | Lucide React |
| **Build** | Vite 7 |

## Project structure

```
src/
├── App.tsx              # Shell: sidebar + sticky header + scroll area + Outlet
├── main.tsx             # React Router config (all routes)
├── components/
│   ├── ui/              # shadcn primitives (button, badge, data-table, etc.)
│   └── *.tsx            # Shared feature components
├── pages/               # One file per screen
├── hooks/               # usePageHeader, useDebouncedSave, use-mobile
└── lib/
    ├── api.ts             # Axios client with JWT interceptor
    ├── auth.ts            # Login/logout API calls
    ├── auth-context.tsx   # AuthProvider + useAuth hook
    ├── workspace-context.tsx # Workspace provider + useWorkspace hook
    ├── services/          # React Query hooks per entity
    │   ├── people.ts
    │   ├── circles.ts
    │   ├── decisions.ts
    │   ├── commitments.ts
    │   ├── sessions.ts
    │   ├── initiatives.ts
    │   ├── projects.ts
    │   └── updates.ts
    └── utils.ts           # cn() classname helper
```

**Convention:** `components/` holds shared components used across multiple pages. If a component is only used by a single page, it should live in a `components/<PageName>/` subfolder. This keeps shared components flat and discoverable while preventing the directory from growing with page-specific concerns.

## Design principles

- **Match Linear's density** — data-dense, keyboard-friendly, fast
- **Light mode only** (V1)
- **shadcn Figma kit** as the design source of truth
- **Inline editing with persistence** — titles, summaries, and properties are editable in place and auto-saved to the backend via PATCH
- **Consistent page headers** — every page injects its header via `usePageHeader` hook into the App shell's sticky header area

## Key architecture decisions

- **React Query for data fetching and mutations** — all API calls go through per-entity hooks in `src/lib/services/*.ts`. Each service exports query hooks (`useDecisions`, `useDecision`) for reading and a mutation hook (`useUpdateDecision`) for writing. Mutations accept a typed `*Patch` payload (e.g. `DecisionPatch`) that uses `*Id` fields for foreign keys (matching the backend's write format).
- **JWT authentication** — `AuthProvider` in `src/lib/auth-context.tsx` manages login/logout state. The Axios client in `src/lib/api.ts` automatically attaches the access token and handles 401 refresh.
- **Multi-workspace context** — `WorkspaceProvider` stores the active workspace slug and exposes it via `useWorkspace()` hook. All API calls include the workspace slug in the URL path.
- **Page header via Outlet context** — pages don't render their own headers. Instead they call `usePageHeader()` which passes header JSX up to the App shell via React Router's Outlet context. This keeps the sticky header outside the scroll area.
- **Detail sidebar is per-page** — each detail page renders its own fixed-position `<aside>`. There is no shared DetailSidebar wrapper component; only the inner `SidebarSection` and `PropertiesRow` primitives are shared.

## Data flow

**Reads:**
```
React component → useDecisions() hook → Axios GET /api/v1/workspaces/{slug}/decisions/
                                       → React Query cache → component re-renders
```

**Writes (inline editing):**
```
Field onChange → optimistic state update (overrides) + mutation call
             → Axios PATCH /api/v1/workspaces/{slug}/decisions/{slug}/
             → onSuccess: invalidate queries → refetch
             → onError: toast.error("Failed to save")
```

For markdown editors (TipTap `onUpdate` fires on every keystroke), saves are debounced via the `useDebouncedSave` hook (800ms delay, flushes on unmount). All other fields (TextField on blur, StatusField/PersonField/DateField on select) save immediately.

The Axios client (`src/lib/api.ts`) handles:
- Attaching the JWT access token to every request
- Automatically refreshing the token on 401 responses
- Redirecting to `/login` if the refresh also fails
